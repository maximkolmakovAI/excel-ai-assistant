# ИИ-ассистент по Excel — Архитектура решения

## Общая структура

Это Streamlit-приложение на Python, которое использует ИИ для анализа данных из Excel-файлов. Пользователь загружает файл, задаёт вопросы на естественном языке, ИИ генерирует и выполняет код для анализа, возвращает результаты текстом, таблицей и графиком.

```
app.py                  # Главный интерфейс Streamlit (вкладки, боковая панель, чат)
data_processor.py       # Загрузка и парсинг Excel-файлов
ai_assistant.py         # Взаимодействие с OpenAI API, генерация и выполнение кода
.streamlit/config.toml  # Конфигурация Streamlit
requirements.txt        # Зависимости проекта
pyproject.toml          # Конфигурация проекта (name, version, dependencies)
.replit                 # Конфигурация Replit (язык, команда запуска)
```

## Файлы проекта

### 1. **app.py** — главное приложение
**Размер:** ~320 строк  
**Язык:** Python 3.11+  
**Назначение:** Streamlit-интерфейс

**Основное содержание:**
- Инициализация состояния сессии (история чата, загруженные данные)
- 3 вкладки:
  - **💬 Чат-аналитик** — диалог с ИИ, отправка вопросов, отображение результатов
  - **🗂️ Данные** — просмотр загруженных листов Excel с поиском и фильтрацией
  - **💡 Примеры вопросов** — 10 готовых вопросов для быстрого тестирования
- Боковая панель:
  - Автозагрузка тестового файла (Сентябрь_2020.xlsx)
  - Загрузка пользовательских файлов
  - Просмотр информации о загруженных листах
  - Переключатели для отладки (показать код, очистить историю)

**Ключевые функции:**
```python
load_sheets(source, name)        # Загрузить и обработать Excel-файл
format_number(n)                 # Форматирование чисел (пробелы как разделители)
init_state()                     # Инициализировать состояние сессии
```

**Зависимости в коде:**
- `streamlit` — UI фреймворк
- `pandas` — работа с данными
- `matplotlib` — отображение графиков
- `ai_assistant` — генерация и выполнение кода
- `data_processor` — парсинг Excel

---

### 2. **data_processor.py** — обработка данных
**Размер:** ~130 строк  
**Язык:** Python 3.11+  
**Назначение:** Загрузка, парсинг и типирование данных из Excel

**Основное содержание:**

**Константы:**
- `SHEET_ALIASES` — альтернативные названия листов (например, "заявки" → "Заявки")
- `COLUMN_TYPES` — правила типизации колонок (date, str, float, int)

**Функции:**
```python
load_excel(file_source)              # Загрузить Excel, вернуть dict DataFrame'ов
  # Входит:
  #   - file_source: путь (str) или BytesIO объект
  # Выход:
  #   - dict: {"Заявки": df, "Заказы": df, "Предприятия": df}
  # Особенности:
  #   - Автоматически находит заголовок (первая непустая строка)
  #   - Удаляет пустые колонки в начале
  #   - Применяет типизацию (дата, число, текст)
  #   - Обрабатывает различные форматы дат (DD.MM.YYYY, YYYY-MM-DD)

get_schema_summary(sheets)           # Описание схемы для ИИ-промпта
  # Вывод: текстовое описание всех листов, колонок и примеров значений

get_sheet_prefix_info(sheets)        # Информация о кодах филиалов
  # Вывод: статистика префиксов номеров заявок (SPB, MSK, EKB и т.д.)
```

**Особенности:**
- Обработка Excel с разными кодировками и форматами
- Автоматическое определение типов данных
- Гибкая обработка дат (поддержка DD.MM.YYYY и других форматов)
- Очистка пустых строк и колонок

### 3. **ai_assistant.py** — ИИ-ассистент
**Размер:** ~190 строк  
**Язык:** Python 3.11+  
**Назначение:** Взаимодействие с OpenAI API, генерация и выполнение аналитического кода

**Основное содержание:**

**Константы:**
```python
MODEL = "gpt-5.2"  # Актуальная модель OpenAI (как на 2025-2026)
SYSTEM_PROMPT      # Большой промпт, инструктирующий ИИ как генерировать код
```

**Функции:**
```python
ask_ai(question, schema_summary, prefix_info, history, sheets)
  # Главная функция для общения с ИИ
  # Входит:
  #   - question: вопрос пользователя (str)
  #   - schema_summary: описание данных (из data_processor)
  #   - prefix_info: информация о филиалах
  #   - history: история последних 8 вопросов (для контекста)
  #   - sheets: dict с DataFrame'ами
  # Выход: dict с полями {text, df, fig, error, code, explanation}

_execute_code(code, sheets)          # Безопасное выполнение сгенерированного кода
  # Входит: Python-код и данные
  # Выход: результат (текст, DataFrame, фигура)
  # Безопасность: ограниченный namespace, только pandas/matplotlib/builtins

fig_to_bytes(fig)                    # Конвертация matplotlib фигуры в PNG
df_to_excel_bytes(df)                # DataFrame → XLSX (xlsxwriter)
df_to_csv_bytes(df)                  # DataFrame → CSV (UTF-8 BOM)
```

**Особенности:**
- Использует OpenAI API через Replit AI Integrations (без собственного ключа)
- ИИ генерирует Pandas-код, который выполняется в безопасном namespace
- Поддерживает контекст диалога (последние 8 вопросов)
- Многоходовые диалоги (уточняющие вопросы)
- Автоматическое форматирование результатов (таблицы, графики, текст)

**SYSTEM_PROMPT (основные правила для ИИ):**
- Переменные: `df_zayavki`, `df_zakazy`, `df_predpriyatiya`
- Установить: `result_text`, `result_df`, `result_fig`
- Matplotlib: использовать ax (оси), figsize=(12,6), не вращать подписи через tick_params()
- Контекст данных о филиалах, структуре листов, кодировании

## Поток данных

```
1. Пользователь загружает Excel
   ↓
2. app.py → data_processor.load_excel()
   ↓
3. Парсинг → DataFrame'ы + схема + информация о филиалах
   ↓
4. Пользователь задаёт вопрос
   ↓
5. app.py → ai_assistant.ask_ai()
   ↓
6. Построение промпта (система, контекст, история, вопрос)
   ↓
7. OpenAI API (gpt-5.2) генерирует Python-код (JSON)
   ↓
8. ai_assistant._execute_code() выполняет в безопасном namespace
   ↓
9. Извлечение результатов (result_text, result_df, result_fig)
   ↓
10. Отображение в Streamlit (текст + таблица + график)
   ↓
11. Сохранение в историю чата
```

## Локальное развёртывание

### Требования
- Python 3.11+
- pip или uv
- Интернет (для OpenAI API)

### Шаги установки

#### 1. Клонирование/копирование файлов
```bash
# Создать папку проекта
mkdir excel-ai-assistant
cd excel-ai-assistant

# Скопировать файлы:
# - app.py
# - data_processor.py
# - ai_assistant.py
# - requirements.txt
# - .streamlit/config.toml
# - attached_assets/Сентябрь_2020_1773221143241.xlsx (опционально, для тестирования)
```

#### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

#### 3. Настройка окружения
Для работы с OpenAI API через Replit AI Integrations нужны переменные окружения:
```bash
export AI_INTEGRATIONS_OPENAI_API_KEY=dummy_key_for_replit
export AI_INTEGRATIONS_OPENAI_BASE_URL=https://your-replit-ai-endpoint
```

**Если развёртываете локально БЕЗ Replit:**
- Либо используйте собственный OpenAI API ключ (замените в ai_assistant.py):
```python
from openai import OpenAI
client = OpenAI(api_key="sk-xxx")  # Ваш ключ
```
- Либо используйте другой LLM (claude, gemini и т.д.)

#### 4. Запуск приложения
```bash
streamlit run app.py
```

Приложение откроется на `http://localhost:8501`

## Ограничения и особенности

### ✅ Что работает хорошо
- Фильтрация данных по разным критериям
- Агрегирование и группировка
- Простые графики (столбчатые, круговые, линейные, гистограммы)
- Многоходовые диалоги с контекстом
- Экспорт в CSV, XLSX, PNG
- Работа с разными Excel-форматами

### ⚠️ Ограничения
- ИИ работает с тем, что видит в схеме данных (если нет колонки — не сможет её использовать)
- Генерируемый код выполняется в узком namespace (только pandas, matplotlib)
- Сложные расчёты могут потребовать 2-3 уточняющих вопроса
- Нет встроенной базы данных (все данные в памяти)

### 🔧 Для продакшена рекомендуется
- Добавить логирование (logging)
- Кэширование результатов (redis)
- Аутентификация пользователей (OAuth, OIDC)
- Ограничение на размер файлов
- Базу данных для сохранения истории анализов

## Примеры вопросов (use-cases)

1. Найти заявки с суммой от 200 тыс. до 1 млн руб.
2. Показать заявки на сумму от 500 тыс. руб., по которым были получены заказы
3. Вывести заявки менеджера на сумму от 1 млн руб.
4. Определить менеджеров московского филиала, у которых количество заявок в первой половине периода было больше, чем во второй
5. Построить круговую диаграмму по количеству заявок, полученных подразделениями
6. Построить гистограмму заявок за период по дням
7. Построить гистограмму с накоплением по менеджерам московского филиала
8. Диаграмма по количеству заявок с сегментами по регионам
